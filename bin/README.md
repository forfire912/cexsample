# CTP 期货交易系统

基于 CTP API 的 Windows GUI 期货交易应用程序，支持交易和实时行情订阅功能。

## 快速开始

### 1. 编译项目
```powershell
.\build.ps1
```

### 2. 运行程序
```powershell
.\run.bat
```

## 项目结构

```
cexsample/
├── api/
│   └── allapi/              # CTP API 统一库（SE版本）
│       ├── *.h              # 头文件（TraderApi & MdApi）
│       ├── *.lib            # 静态库
│       └── *.dll            # 动态库
├── bin/                     # 可执行文件目录
│   ├── CTP_Trader.exe       # 主程序
│   └── *.dll                # 运行时依赖DLL
├── src/                     # 源代码
│   ├── main.c               # GUI界面和主逻辑
│   ├── ctp_trader.cpp       # CTP API封装
│   └── ctp_trader.h         # 接口定义
├── flow/                    # CTP流文件（自动生成）
├── obj/                     # 编译中间文件
├── build.ps1                # 编译脚本（推荐）
└── run.bat                  # 运行脚本
```

## 技术架构

### API 集成
- **TraderApi**：交易接口（认证、登录、查询、报单）
- **MdApi**：行情接口（连接、订阅、实时推送）

### 编译环境
- **编译器**：Visual Studio 2022 MSVC (x64)
- **语言**：C/C++ 混编（UNICODE模式）
- **UI框架**：Win32 API
- **库文件**：
  - `thosttraderapi_se.lib` (3.5MB)
  - `thostmduserapi_se.lib` (3.1MB)

### 编译过程
```
[1/4] 清理旧文件
[2/4] 编译 C++ 文件 (ctp_trader.cpp)
[3/4] 编译 C 文件 (main.c)
[4/4] 链接生成可执行文件
[5/5] 复制 DLL 到 bin 目录
```

## 常见问题

### 1. 杀毒软件误报
**问题**：Windows Defender 删除 `CTP_Trader.exe`

**解决方案**：
```powershell
# 以管理员身份运行
.\add_exclusion.ps1
```

或手动添加排除项：
- Windows 安全中心 → 病毒和威胁防护 → 管理设置 → 排除项
- 添加文件夹：`D:\projects\other\cexsample\bin`

### 2. 编译失败
**问题**：找不到 `vcvars64.bat`

**解决方案**：
- 确保安装了 Visual Studio 2022
- 检查路径：`C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvars64.bat`
- 修改 `build.ps1` 中的路径

### 3. 行情无数据
**问题**：订阅行情后界面无反应

**原因**：不在交易时段，服务器不推送数据

**解决方案**：
- 在交易时段测试（09:00-15:00 或 21:00-02:30）
- 查看 `ctp_debug.log` 确认订阅成功
- 确保行情前置地址正确

### 4. 连接失败
**检查项**：
- 网络连接是否正常
- 前置地址格式：`tcp://IP:PORT`
- 券商代码、用户名、密码是否正确
- 查看 `flow/` 目录下的错误日志

## 调试信息

# CTP 交易客户端示例

这是一个基于 CTP API 的简易 Windows 交易客户端，用于演示如何连接交易和行情服务器，并进行基本的查询和行情订阅操作。

## 软件功能

-   连接到 CTP 交易和行情服务器。
-   支持主席-SimNow 模拟环境。
-   查询当日委托。
-   查询当前持仓。
-   查询可用合约。
-   订阅和显示单个合约的实时行情。
-   所有界面和状态信息均已中文化。

## 编译与运行

### 环境要求

-   Windows 操作系统
-   Visual Studio 2019 或更高版本（需要 C++ 桌面开发环境）

### 编译步骤

项目已包含一个 PowerShell 编译脚本 `build.ps1`，它会自动查找 Visual Studio 环境并执行编译。

1.  打开 PowerShell 终端。
2.  进入项目根目录。
3.  执行编译脚本：
    ```powershell
    .\build.ps1
    ```
4.  如果编译成功，会在 `bin/` 目录下生成 `CTP_Trader.exe`。

### 运行程序

直接双击运行 `bin/CTP_Trader.exe`，或者使用项目根目录下的 `run.bat` 脚本启动。

## 使用说明

### 1. 连接交易服务器

程序启动后，首先需要连接到交易服务器。

1.  **经纪商ID**: 填入期货公司的 Broker ID（例如，SimNow 为 `9999`）。
2.  **用户ID**: 填入你的交易账户 ID。
3.  **密码**: 填入交易密码。
4.  **交易前置**: 填入交易服务器的地址（`tcp://...`）。
5.  **认证码**: 填入期货公司提供的 AppID 对应的认证码（Auth Code）。
6.  点击 **连接登录** 按钮。
7.  观察底部 **状态栏** 的提示，如果显示“登录成功！”，则表示交易服务器已连接。

### 2. 连接行情服务器

交易服务器登录成功后，可以连接行情服务器以接收实时数据。

1.  **行情前置**: 确认行情服务器地址无误。
2.  点击 **连接行情** 按钮。
3.  观察状态栏提示，如果显示“行情登录成功！”，则表示行情服务器已连接。

### 3. 查询功能

连接成功后，可以使用查询功能获取账户信息。

-   **查询委托**: 点击 **查询委托** 按钮，下方的表格将显示当日所有的委托记录。
-   **查询持仓**: 点击 **查询持仓** 按钮，表格将显示当前账户的所有持仓信息。
-   **查询合约**: 点击 **查询合约** 按钮，表格将显示所有可交易的合约列表。你也可以在 **合约代码** 输入框中填入特定品种（如 `rb`）来查询该品种下的所有合约。

### 4. 订阅实时行情

行情服务器连接成功后，可以订阅单个合约的实时行情。

1.  在 **合约代码** 输入框中，填入你想要订阅的合约代码（例如 `rb2410`）。
2.  点击 **订阅行情** 按钮。
3.  订阅成功后，下方的表格会自动切换并开始显示该合约的实时五档行情和关键价格信息。

## 注意事项

-   所有操作的结果和状态都会在主窗口底部的状态栏实时更新，请留意提示信息。
-   查询合约时，由于合约数量巨大（数万条），程序默认最多只显示前 500 条，以防止界面卡死。完整的合约数量会在状态栏提示。
-   请在期货市场的交易时段进行测试，否则可能无法收到实时行情数据。
-   日志文件 `ctp_debug.log` 会记录程序运行过程中的关键 API 调用和状态，便于排查问题。
